# Engine Yard’s Early Support for GitHub

In GitHub’s early days (circa 2008–2009), Engine Yard played a critical role by subsidizing GitHub’s infrastructure and co-marketing efforts. Engine Yard provided GitHub with free hosting on a dedicated Rails cluster, effectively eliminating GitHub’s largest early expense. In April 2008 Engine Yard publicly announced it was "sponsoring" the then-new GitHub service by hosting both GitHub and the Lighthouse bug-tracker on Engine Yard’s scalable Ruby-on-Rails platform【18†L27-L33】. Chris Wanstrath (GitHub co-founder) himself thanked Engine Yard, noting GitHub’s code and tickets were "safely and securely hosted" on Engine Yard’s generous cluster【14†L487-L495】. In practice this meant GitHub ran on Engine Yard’s servers for free – only in exchange for publicity (GitHub banners and blog mentions). Tom Preston-Werner (GitHub co-founder) later summed it up: "We had a partnership with Engine Yard. They gave us free hosting and we gave them publicity"【31†L372-L375】.

- Free Infrastructure ("value-swap" hosting): Engine Yard gave GitHub free server hosting, waiving the site’s hosting bills. As GitHub’s scale grew, Preston-Werner explained this was done via a "value swap" – Engine Yard covered GitHub’s hosting, and in return GitHub gave Engine Yard publicity and free user accounts for Engine Yard customers【5†L368-L377】【31†L372-L375】. (GitHub’s users still remembered this deal fondly – a GitHub-blog summary notes Engine Yard’s free hosting "eliminated the single biggest early cost"【26†L294-L299】.) Data-center reports confirm Engine Yard specialized in Rails apps and rooted this arrangement in the Ruby community – they later admitted they began moving GitHub off free hosting only because GitHub was attracting non-Ruby users (making subsidizing it less attractive)【5†L368-L377】.

- Dedicated Cluster Support: Early on, GitHub ran on Engine Yard’s own "private cluster." A joint press release (covered by LWN) announced that GitHub (and Lighthouse) would run on Engine Yard’s infrastructure【18†L27-L33】. GitHub’s engineers noted that Engine Yard’s cluster was generous, keeping GitHub’s servers secure and fast【14†L487-L495】. This direct support let the small GitHub team scale from a shared Slicehost box to a full cluster of machines without raising cash. One observer recalls that moving GitHub from a shared slice to Engine Yard’s dedicated servers was a "big deal" that saved GitHub "a couple hundred thousand dollars in hosting and scaling costs"【31†L372-L375】.

- Co-Marketing and Community Ties: The partnership was very public. For example, at RailsConf 2008 GitHub’s founders even placed a large banner in the Engine Yard booth【36†L489-L492】. Engine Yard employees were active in the Ruby/Rails community, helping seed GitHub’s user base. (Engine Yard co-developed the Merb framework, and Merb was reportedly "the first big project" to host its repository on GitHub, leveraging that community connection【23†L476-L483】.) In blog posts, GitHub’s team enthusiastically endorsed Engine Yard. Chris Wanstrath wrote "If you’re looking for awesome, hands-on hosting, we highly recommend Engine Yard"【14†L487-L495】. This co-promotion (GitHub mentions Engine Yard in blogs and at events, Engine Yard hosts GitHub’s site) exemplified the "win/win" exchange they had. As Preston-Werner noted, such partnerships "save a bunch of money" and are "key to bootstrapping a startup"【31†L372-L375】.

- Ruby Community Network: Beyond infrastructure, Engine Yard’s leadership and community network provided indirect support. As Ruby hosting experts, they signaled trust in GitHub to other Rails developers. GitHub’s early growth owed much to Rubyists spreading the word; Chris Wanstrath later acknowledged that beyond Engine Yard’s help, "key members of the Ruby and JavaScript communities" were needed for GitHub’s success【3†L317-L319】. By backing GitHub, Engine Yard helped legitimize it among Ruby developers.

In short, Engine Yard’s early involvement kept GitHub alive and growing with almost no cash outlay. The free servers and mutual publicity essentially bought GitHub valuable runway until its user base—and revenue—took off. Both sides have publicly credited the partnership: when GitHub eventually outgrew Engine Yard (moving to Rackspace in 2009), co-founder Tom Preston-Werner thanked Engine Yard "for getting us to where we are"【5†L392-L396】. Likewise, Engine Yard’s CTO noted they were proud to support GitHub when it was still a scrappy startup. Without Engine Yard’s hosting and community support, GitHub "would probably look like a neat website and not a profitable company," as Wanstrath admitted【3†L317-L319】.
