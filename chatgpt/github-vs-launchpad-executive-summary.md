# GitHub vs. Launchpad: Executive Summary

- Origins & Founders: GitHub launched in April 2008, founded by Tom Preston-Werner, Chris Wanstrath, P.J. Hyett and Scott Chacon【80†L299-L302】, to make Git-based development collaborative. Launchpad was created a few years earlier (circa 2004–2005) by Canonical Ltd. (Mark Shuttleworth’s Ubuntu company) to support Ubuntu and related projects【57†L13-L17】【81†L685-L693】.

- Funding & Ownership: GitHub was initially bootstrapped, then raised venture capital (e.g. $100M in 2012【38†L128-L136】, $250M in 2015【41†L107-L115】) before being acquired by Microsoft for \$7.5 billion in 2018【32†L75-L83】. Launchpad has always been funded internally by Canonical; it is owned and governed by Canonical Ltd. (a private company). It was open-sourced under the AGPL in 2009【81†L685-L693】, but Canonical retains control and provides any paid private hosting.

- Organization & Governance: GitHub, Inc. is a subsidiary of Microsoft (since 2018【32†L75-L83】) with ~5,600 employees (2024)【80†L255-L262】. Launchpad development is handled by Canonical’s web engineering teams (with contributions from volunteers); exact staffing is not public but it is a Canonical project. GitHub’s leadership has included CEOs Wanstrath (co-founder), Nat Friedman (Microsoft hire), and now Thomas Dohmke. Launchpad’s direction is overseen by Canonical’s management (Mark Shuttleworth, Jane Silber, etc.).

- Community & Contributions: GitHub is the dominant global code-hosting platform. By 2025 it hosts ~180 million developers and 630 million repositories【52†L611-L619】, and supports millions of corporate and open-source contributors worldwide. Launchpad’s user base is much smaller and focused: it hosts tens of thousands of projects (around 40,000 by 2018【17†L179-L182】), chiefly Ubuntu and related open-source communities (Ubuntu, Bazaar, Linux Mint, Upstart, OpenStack, etc. use Launchpad)【17†L208-L218】. GitHub’s governance is largely "benevolent dictatorship" by repo owners, while Launchpad’s governance is Canonical-driven (with some community input via merge proposals and patch contributions).

- Monetization: GitHub operates on a freemium model: free for public (open source) use, with paid plans for private or enterprise use (Team, Enterprise Cloud/Server) and revenue from services (Marketplace fees, GitHub Sponsors commissions). By 2022 GitHub’s revenue was roughly \$1 billion annually【80†L339-L342】. Launchpad is free for qualifying open-source projects【81†L725-L729】, with fees only for private or commercial use (though in practice few customers use that). Unlike GitHub, Launchpad itself does not generate significant independent revenue; Canonical’s income comes from Ubuntu support and services, not directly from Launchpad.

- User Experience & Workflows: GitHub’s web UI and Git tooling are widely regarded as slick and modern: key features include Git-based code hosting, Pull Requests for code review, Issues for tracking bugs, GitHub Actions for CI/CD, GitHub Packages (npm, Docker, etc.), a large Apps/Marketplace ecosystem, and integrations (e.g. VS Code, IDEs). Launchpad offers a broader suite: Bazaar (and now Git) code hosting with merge proposals for reviews, a unique multi-project bug tracker, a translations portal (Rosetta), a Q&A forum (Answers), and package build/PPAs (Soyuz). Its interface is older and more complex; tasks like code review and CI are less streamlined than GitHub’s. Launchpad’s workflows emphasize cross-project coordination (e.g. sharing bugs across Ubuntu releases), while GitHub focuses on individual repositories and integrated tooling.

- Scale & Growth: GitHub has enjoyed explosive growth. As of 2025, >180 million developers actively use GitHub to collaborate on over 630 million repos【52†L611-L619】. In 2025 alone, 36+ million new users joined (driven by GitHub Copilot’s free tier)【52†L559-L568】. Development activity surged (230 repos created per minute; ~1 billion commits in 2025)【52†L499-L508】. Launchpad’s growth has been far more modest: it grew to tens of thousands of projects (about 40k by 2018【17†L179-L182】) and serves the Ubuntu ecosystem (Ubuntu and many spin-offs). - Security, IP & Legal: GitHub is a proprietary platform but obligates users to own/licence their code; users choose open-source licenses, and GitHub provides automated license detection. GitHub has robust security features (Dependabot alerts, token scanning, etc.) and a DMCA takedown process. Notable incidents include a massive DDoS attack in Feb 2018【80†L377-L378】, but no large-scale security breaches. Launchpad’s code is open-source (AGPL)【81†L685-L693】. Content uploaded to Launchpad (code, bug comments) is copyrighted by contributors, under project licenses. Launchpad’s terms explicitly state it is free for open-source projects with OSI/DFSG-compliant licenses【60†L762-L771】. Canonical reserves the right to remove content that violates laws or terms, but Launchpad has not had major public legal controversies (unlike GitHub’s occasional DMCA disputes or content moderation debates).

- Culture & Controversies: GitHub’s culture is open and collaborative but occasionally controversial. For example, in 2014 CEO Tom Preston-Werner resigned amid internal harassment allegations【80†L313-L320】. The platform also faces community scrutiny over policies (e.g. license of Copilot’s training data, DMCA takedowns, or Microsoft’s ownership concerns). GitHub’s community includes many corporate and individual contributors. Launchpad’s culture is that of the broader Ubuntu community. It faced criticism early on for being closed-source internally (Canonical gradually open-sourced it starting 2007 and fully by 2009【81†L685-L693】). Some users complain that Launchpad’s interface is clunky or buggy, but controversies have been minor. Launchpad is mainly used by free-software developers (Ubuntu translators, packagers, etc.) who value its cross-project coordination.

- Strategic Position & Outlook: GitHub’s strength is its massive network effect, rich feature set, and integration with Microsoft’s cloud/AI strategy. It dominates open-source hosting and continues investing in AI-powered development (e.g. Copilot expansion, GitHub Actions AI, security Scanning, etc.)【52†L513-L519】【71†L169-L178】. Weaknesses include its proprietary nature, reliance on Microsoft, and complexity for new users. Launchpad’s strength lies in its end-to-end support for Linux distro development (code, QA, translations, PPAs). Its niche focus (Ubuntu ecosystem) and integrated bug/translation tools are unique. Weaknesses are limited adoption outside Canonical projects, reliance on Bazaar (now partly supplanted by Git), and a dated UI. Canonical is revamping Launchpad’s UI (new Ubuntu series page in 2026【57†L19-L27】) but Launchpad faces a general trend toward Git/GitHub, Snap packaging, and other tools. The future for GitHub is continued growth with AI and cloud services【52†L523-L532】【71†L169-L178】, whereas Launchpad’s future is modernization (UX improvements) and sustained support by Canonical for Ubuntu-related workflows.
